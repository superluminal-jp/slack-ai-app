# Implementation Plan: Verification Zone Usage History

**Branch**: `039-usage-history` | **Date**: 2026-03-16 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/039-usage-history/spec.md`

## Summary

Append-only usage history for every Verification Agent request. **機密性保持のため入出力テキストは DynamoDB に格納せず S3 に分離する。** DynamoDB はメタデータ・インデックスのみ保持し、入出力テキストと添付ファイルはすべて専用 S3 バケットに書き込む。Integrates as the final fail-open step in `pipeline.py` — write failure never blocks user response.

## Technical Context

**Language/Version**: Python 3.11 (`python:3.11-slim`, ARM64) + TypeScript 5.x (CDK)
**Primary Dependencies**: boto3 ~=1.42.0, aws-cdk-lib (existing), strands-agents[a2a,otel] ~=1.25.0
**Storage**: DynamoDB (new `usage-history` table) + S3 (new `usage-history` bucket)
**Testing**: pytest (Python), Jest (CDK)
**Target Platform**: Bedrock AgentCore Runtime (verification zone)
**Project Type**: Single agent zone addition within `verification-zones/verification-agent/`
**Performance Goals**: Write overhead < 50 ms; does not block response path (fail-open)
**Constraints**: 10 MB per attachment (inherited from existing pipeline limit); text content has no size constraint in S3
**Scale/Scope**: Append-only writes, ~1 write/request; read path is administrative only

## New Module: `usage_history.py`

```python
@dataclass
class PipelineResult:
    existence_check: bool = True
    authorization: bool = True
    rate_limited: bool = False
    rejection_stage: str = ""
    rejection_reason: str = ""

@dataclass
class OrchestrationResult:
    agents_called: list[str] = field(default_factory=list)
    turns_used: int = 0
    success: bool = True

@dataclass
class UsageRecord:
    channel_id: str
    correlation_id: str
    team_id: str = ""
    user_id: str = ""
    input_text: str = ""   # written to S3 only; NOT stored in DynamoDB
    output_text: str = ""  # written to S3 only; NOT stored in DynamoDB
    pipeline_result: PipelineResult = field(default_factory=PipelineResult)
    orchestration: Optional[OrchestrationResult] = None
    duration_ms: int = 0
    attachment_keys: list[str] = field(default_factory=list)  # temp S3 keys to copy
    skipped_attachments: list[dict] = field(default_factory=list)
    ttl_days: int = 90

def save(
    table_name: str,
    history_bucket: str,
    temp_bucket: str,
    record: UsageRecord,
) -> None:
    """
    Write usage history. Fail-open.
    1. s3.put_object() content/…/input.json  (if input_text non-empty)
    2. s3.put_object() content/…/output.json (if output_text non-empty)
    3. s3.copy_object() per attachment key
    4. dynamodb.put_item() metadata + s3_content_prefix (NO input_text/output_text)
    """
```

## CDK New Constructs

### `usage-history-table.ts`
- DynamoDB table: `{stackName}-usage-history`
- PK: `channel_id` (STRING), SK: `request_id` (STRING)
- GSI `correlation_id-index`: PK=`correlation_id`
- TTL: `ttl`, PAY_PER_REQUEST, AWS_MANAGED, DESTROY

### `usage-history-bucket.ts`
- S3 bucket: `{stackName.toLowerCase()}-usage-history`
- SSE-S3, enforceSSL, BlockPublicAccess.BLOCK_ALL, DESTROY, autoDeleteObjects
- Lifecycle rule: `content/` prefix, expiration 90 days
- Lifecycle rule: `attachments/` prefix, expiration 90 days

## CDK Stack Wiring (`verification-stack.ts`)

```typescript
const usageHistoryTable = new UsageHistoryTable(this, "UsageHistoryTable");
const usageHistoryBucket = new UsageHistoryBucket(this, "UsageHistoryBucket");

// In VerificationAgentRuntime props:
usageHistoryTable.table  → new prop usageHistoryTable
usageHistoryBucket.bucket → new prop usageHistoryBucket

// In runtime construct: IAM grants
usageHistoryTable.table.grantWriteData(this.executionRole);
usageHistoryBucket.bucket.grantPut(this.executionRole, "content/*");      // テキスト書き込みのみ
usageHistoryBucket.bucket.grantPut(this.executionRole, "attachments/*");  // 添付ファイル書き込みのみ

// Env vars injected:
USAGE_HISTORY_TABLE_NAME: usageHistoryTable.table.tableName
USAGE_HISTORY_BUCKET_NAME: usageHistoryBucket.bucketName
```

## Key Implementation Rules

1. **Fail-open**: `save()` wraps everything in `try/except Exception` → log WARNING, never raise
2. **Text → S3 only**: `input_text` and `output_text` are written as `content/.../input.json` and `content/.../output.json` via `s3.put_object()`; they are **never written to DynamoDB**
3. **DynamoDB → metadata only**: DynamoDB item contains `s3_content_prefix` (pointer) but no text content
4. **Attachment copy**: `s3.copy_object(CopySource={Bucket: temp_bucket, Key: temp_key}, Bucket: history_bucket, Key: history_key)` — no re-download
5. **Temp-key mapping**: `s3_file_manager.py` already stores attachment keys; pass them through pipeline context
6. **correlation_id fallback**: If absent, generate UUID and log WARNING
7. **Input text source**: from `OrchestrationRequest.user_text` (post-enrichment full text)
8. **Output text source**: from `OrchestrationResult.synthesized_text` (final agent response)
9. **Absent content**: If pipeline rejects before enrichment, `input_text` is empty → skip `input.json` write. If orchestration not run, `output_text` is empty → skip `output.json` write.

## Dependencies & Risks

| Item | Risk | Mitigation |
|---|---|---|
| S3 PutObject latency for text | Additional S3 writes per request (input + output) | Post-response; not on critical path. Fail-open means timeout doesn't affect user |
| Temp bucket key availability | `s3_file_manager.py` keys must be accessible in pipeline context | Pass `temp_attachment_keys` list through `pipeline.py` run context (already available) |
| CDK construct additions | Adding 2 constructs + 3 IAM grants | Minimal risk; follows existing patterns exactly |
| S3 content orphan on DynamoDB failure | Text written to S3 but DynamoDB PutItem fails → orphan content files | Acceptable; TTL lifecycle will clean up; DynamoDB failure is logged |
