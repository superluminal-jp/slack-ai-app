# Data Model: DynamoDB Agent Registry

**Feature**: 055-dynamodb-agent-registry
**Date**: 2026-03-25

## Entity: Agent Registry Entry

Represents a registered execution agent in the system.

### Table: `{stackName}-agent-registry`

| Attribute | Type | Key | Description |
|-----------|------|-----|-------------|
| `env` | String | PK (Partition Key) | Environment identifier: "dev" or "prod" |
| `agent_id` | String | SK (Sort Key) | Unique agent name: "time", "docs", "fetch-url", "file-creator", "slack-search" |
| `arn` | String | — | AgentCore Runtime ARN |
| `description` | String | — | Human-readable agent purpose |
| `skills` | List of Map | — | Capability descriptors (each: `id`, `name`, `description`) |
| `updated_at` | String | — | ISO 8601 timestamp of last registration |

### Access Patterns

| Pattern | Operation | Key Condition | Frequency |
|---------|-----------|---------------|-----------|
| Load all agents for env | Query | `env = :env_value` | At agent startup |
| Refresh registry | Query | `env = :env_value` | On demand (rare) |
| Register agent | PutItem | `env` + `agent_id` | At deploy time only |

### Table Settings

- **Billing**: PAY_PER_REQUEST (on-demand) — write frequency is deploy-time only
- **Encryption**: AWS_MANAGED
- **Removal Policy**: DESTROY (table is regenerated by deploy scripts)
- **No TTL**: Entries are overwritten at deploy, not expired
- **No PITR**: Data is ephemeral and regenerated
- **No GSI**: Single access pattern (Query on PK)

### Skill Sub-Entity

Each skill within an agent entry:

| Field | Type | Description |
|-------|------|-------------|
| `id` | String | Skill identifier (e.g., "get-current-time") |
| `name` | String | Display name (e.g., "Get Current Time") |
| `description` | String | What the skill does |

### Example Item

```json
{
  "env": { "S": "dev" },
  "agent_id": { "S": "time" },
  "arn": { "S": "arn:aws:bedrock-agentcore:ap-northeast-1:123456789012:runtime/SlackAI_Time_Dev" },
  "description": { "S": "Returns the current time in the requested timezone" },
  "skills": { "L": [
    { "M": {
      "id": { "S": "get-current-time" },
      "name": { "S": "Get Current Time" },
      "description": { "S": "Returns the current time in a specified timezone" }
    }}
  ]},
  "updated_at": { "S": "2026-03-25T10:30:00+09:00" }
}
```

### Validation Rules

- `env` must be non-empty string ("dev" or "prod")
- `agent_id` must be non-empty string matching known agent names
- `arn` must be non-empty string (ARN format)
- `description` must be non-empty string
- `skills` may be empty list
- `updated_at` must be ISO 8601 string
