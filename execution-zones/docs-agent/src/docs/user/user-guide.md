# ユーザーガイド

**目的**: ボットの基本的な使い方、応答の流れ、対応機能、効果的な質問のコツをエンドユーザーに伝える。
**対象読者**: エンドユーザー（Slack で AI ボットを利用する方）
**最終更新日**: 2026-02-14

---

## ボットの呼び出し方

1. **メンションする**: チャンネルまたはダイレクトメッセージで、`@AIアプリ名` とメンションします（アプリ名は組織で設定された名前です）。
2. **質問や依頼を書く**: メンションのあとに、質問や依頼の内容を続けて入力します。
3. **送信**: Enter キーで送信します。

**例**:
```
@AIアプリ名 明日の会議の議題を3つ提案して
```

メンションと質問を 1 つのメッセージで送るだけで、AI に依頼できます。別のツールを開く必要はありません。

---

## 何が起きるか（応答の流れ）

- **すぐに「処理中」**: 送信後、通常 **2 秒以内** に「処理中です... 少々お待ちください」というメッセージが表示されます。これでリクエストが受け付けられたことが分かります。
- **回答はスレッドに表示**: AI の回答は、あなたが送ったメッセージへの**スレッド返信**として表示されます。チャンネル全体に新しいメッセージが立つわけではありません。スレッドを開くと回答が読めます。
- **回答までの時間**: 「処理中」のあと、実際の回答が表示されるまでには、質問の内容や長さ、システムの負荷によって **数秒〜数十秒** かかることがあります。その間は他の作業を続けて大丈夫です。

| タイミング | あなたが見るもの |
|------------|------------------|
| 送信直後 | 自分のメッセージが投稿される |
| 2 秒以内 | 「処理中です...」の応答 |
| 処理完了後 | スレッド内に AI の回答が表示される |

## 使える機能

### テキストでの質問

- 一般的な質問（例: 用語の説明、アイデアの整理）
- 文章の要約や言い換え
- コードやデータ分析の依頼（組織で許可されている場合）

### 画像の分析

- 画像を添付し、「この画像について説明してください」のように依頼できます。
- スクリーンショットの説明、図表の内容の要約などに使えます。
- **対応形式**: PNG, JPEG, GIF, WebP（**最大 10MB**）

### ドキュメントの処理

- PDF、Word、Excel、PowerPoint、テキストファイルなどを添付し、要約や質問への回答を依頼できます。
- **対応形式**: PDF, DOCX, CSV, XLSX, PPTX, TXT（**最大 5MB**）

### スレッドでの会話の続き

- 同じスレッド内で追加の質問を送ると、それまでの会話の流れを踏まえた返答が得られます。
- 例: 最初に「〇〇の特徴を教えて」と聞き、そのあとスレッドで「さっき言った 2 番目をもう少し詳しく」と続けて質問できます。

---

## 効果的な質問のコツ

- **はっきり書く**: 何を知りたいか、何をしてほしいかを具体的に書くと、より役立つ回答が得られやすくなります。
- **必要なら背景を添える**: 「〇〇のプロジェクトで使う前提で」のように、文脈を一言添えると伝わりやすくなります。
- **長すぎる質問は避ける**: 非常に長い文章や、一度にたくさんの質問を詰め込むと、処理に時間がかかったり、回答が分かりにくくなることがあります。必要なら質問を分けるとよいです。
- **スレッドを活用する**: フォローアップは同じスレッドで続けると、会話の流れが保たれます。

---

## よくあるメッセージと対処

- **「処理がタイムアウトしました」**: 処理に時間がかかりすぎた場合です。質問を短くするか、しばらくしてからもう一度試してください。
- **「画像が大きすぎます」**: 画像は 10MB までです。圧縮するか、より小さい画像を送ってください。
- **「このファイル形式には対応していません」**: 画像は PNG/JPEG/GIF/WebP、ドキュメントは PDF/DOCX/CSV/XLSX/PPTX/TXT のみ対応しています。形式を確認して送り直してください。
- **「画像のダウンロードに失敗しました」**: ボットがそのチャンネルに参加しているか、画像が削除されていないか確認してください。

その他の使い方や禁止事項は [利用ポリシー](./usage-policy.md) を、よくある質問は [FAQ](./faq.md) を参照してください。
